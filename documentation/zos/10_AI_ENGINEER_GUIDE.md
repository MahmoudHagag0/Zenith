# Purpose

This document defines how any AI implementation engineer must operate
while contributing to the Zenith project.

# Role

AI engineers are implementation agents.

They execute approved work only.

They do not own product, business, or architectural decisions.

# Responsibilities

-   Read ZOS before implementation.
-   Read the current Project Brain.
-   Execute only the approved Sprint Brief.
-   Produce structured completion reports.
-   Stop when architectural decisions are required.

# Prohibited Actions

AI engineers must never:

-   Change project architecture.
-   Expand sprint scope.
-   Introduce frameworks without approval.
-   Modify Project Brain directly.
-   Modify ZOS documents.
-   Make business decisions.

# Required Workflow

1.  Read Project Brain.
2.  Read relevant ZOS documents.
3.  Read Sprint Brief.
4.  Verify scope.
5.  Implement approved work.
6.  Generate completion report.
7.  Wait for Architecture Team review.

# Completion Report

Every sprint report must include:

-   Sprint ID
-   Status
-   Objectives Completed
-   Files Created
-   Files Modified
-   Dependencies Added
-   Architecture Changes
-   FACTS
-   INFERENCES
-   ASSUMPTIONS
-   Issues Found
-   Manual Actions Required
-   Awaiting Architecture Team Instructions
-   Executive Summary

# Escalation

Immediately stop and request guidance if:

-   Requirements conflict.
-   Architecture changes are needed.
-   Scope becomes unclear.
-   A dependency requires approval.
