# Objective

Define the official engineering lifecycle used by the Zenith project.

# Workflow

1.  Architecture Planning
2.  Sprint Brief Approval
3.  Implementation
4.  Engineering Completion Report
5.  Architecture Review
6.  Approval or Required Changes
7.  Project Brain Update
8.  Sprint Closure

# Responsibilities

## Architecture Team

-   Define architecture
-   Approve sprint scope
-   Review implementation
-   Maintain ZOS

## Implementation Engineer (AI/Human)

-   Execute only approved work
-   Never change architecture
-   Produce completion report
-   Wait for review

# Definition of Done

A sprint is complete only when: - Scope implemented - No unauthorized
changes - Report submitted - Architecture review completed - Project
Brain updated - Sprint formally closed

# Escalation Rules

Stop and request Architecture Team guidance when: - New dependency is
required. - Architecture change is proposed. - Requirements are
unclear. - Scope expansion is requested.

# Deliverables

Every sprint must produce: - Source code - Updated documentation (if
applicable) - Completion report - Final assessment
