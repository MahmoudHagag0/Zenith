# Objective

Build the Narrative Composer — `25_PRODUCT_BLUEPRINT.md` §10's Implementation
Order step 2 — the deterministic, template-based synthesis layer that turns
the Confluence Engine Consumer's already-structured output (S1-019) into a
Morning Brief narrative satisfying Constitution §12.1 (Story Before Chart)
and the ten Product Rules (§13). This is explicitly **not** the generative
AI Reasoning Layer (`25_PRODUCT_BLUEPRINT.md` §6, §8) — every sentence this
Sprint produces is a deterministic function of fields S1-019 already
computes; nothing is generated by a model, and nothing is asserted that
is not directly traceable to an existing `InstrumentReading`/
`DecisionCenterResponse` field.

---

# Approved Scope

1. **`morning-brief.types.ts`** — `MorningBriefResponse`, `MorningBriefEntry`
   DTOs. Bounded to the top `5` ranked opportunities (`MAX_MORNING_BRIEF_ENTRIES`,
   Missing Decision), reusing `DecisionCenterResponse.opportunities`'s own
   existing rank order verbatim (no re-ranking).
2. **`narrative-composer.util.ts`** — pure, deterministic template functions
   composing, per entry: a **story** sentence (direction + supporting
   evidence-count, leading the sentence — Constitution §12.1), a **why**
   sentence (quoting the lead contributor's own `interpretationSummary`
   verbatim — S1-019's existing, already-selected highest-`INTERPRETATION`-
   confidence contributor, never re-selected here), a **confidence**
   explanation (quoting `interpretationConfidence`/`methodologyConfidenceCeiling`
   and their own existing `.explanation` text — Constitution §12.6, never a
   bare number), and an **uncertainty** explanation (quoting `Limitations`,
   disclosed `disagreementDimensions`, and non-participating Provider count
   — Constitution §12.7). A session-level **headline** and an explicit
   **No Trade** narrative (Product Rule 9, Constitution §12.4) for
   `NO_CLEAR_OPPORTUNITY`, distinct in wording from `DEGRADED` ("unable to
   compute" vs. "answered: nothing qualifies" — the same distinction
   S1-019 already established at the readiness-state level, carried
   through to narrative language here). All language follows
   `27_ZENITH_EXPERIENCE_LANGUAGE.md` §8 (never exciting/dramatic/
   persuasive/predictive; always calm/evidence-based/professional).
3. **`NarrativeComposerService`** — thin injectable wrapper around Scope
   item 2's pure functions (mirroring `DashboardService`'s own use of
   `net-direction-ranking.util.ts`).
4. **`MorningBriefService`** — orchestrator: calls the existing
   `DashboardService.getDecisionCenter(userId)` (S1-019, unmodified) and
   passes its result directly to `NarrativeComposerService.compose()`.
   **No new data-fetching, no new Confluence/Provider interaction, no
   re-implementation of instrument-gathering or ranking** — 100% reuse of
   S1-019's own orchestration.
5. **`MorningBriefController`** — `GET /morning-brief`, JWT-authenticated,
   trader-scoped, mirroring `DashboardController`'s own exact pattern.
6. **`MorningBriefModule`** — imports `DashboardModule`. Requires
   `DashboardModule` to additionally export `DashboardService` (currently
   exports only `InstrumentReadingService`) — a one-line, additive change,
   not a redesign.
7. **Comprehensive tests**: deterministic template-output assertions for
   every input combination (full participation, partial participation,
   disagreement present, zero contributors, `NO_CLEAR_OPPORTUNITY`,
   `DEGRADED`, zero tracked instruments), plus a delegation test proving
   `MorningBriefService` performs no independent ranking/aggregation (it
   calls `DashboardService.getDecisionCenter()` exactly once and passes
   its exact return value to the composer).

# Out of Scope

- **No generative/conversational AI** — the AI Reasoning Layer remains a
  separate, later, P1 component (`25_PRODUCT_BLUEPRINT.md` §6, §8).
- **No UI, React, Flutter, styling, or frontend code** — backend only.
- **No new Confluence Engine, Analysis Provider, or Execution Engine
  change** — this Sprint composes only S1-019's own existing output.
- **No re-ranking or re-aggregation of instruments** — `MorningBriefService`
  reuses `DashboardService.getDecisionCenter()`'s own result verbatim;
  it never recomputes readiness, net direction, or relevance score.
- **No Calendar/News or Trading Journal continuity input** — both are
  optional/P1 inputs per `25_PRODUCT_BLUEPRINT.md` §2 (Morning Brief) and
  neither area exists yet.
- **No modification to `24_ZENITH_PRODUCT_CONSTITUTION.md`,
  `25_PRODUCT_BLUEPRINT.md`, `26_DASHBOARD_HOME_SPECIFICATION.md`, or
  `27_ZENITH_EXPERIENCE_LANGUAGE.md`** — all four remain frozen, cited only.

# Missing Decisions (Anticipated Implementation-Time Calibration)

1. **Bounded entry count** (`MAX_MORNING_BRIEF_ENTRIES`) — how many ranked
   opportunities appear in one Morning Brief.
2. **Instrument scope**: `25_PRODUCT_BLUEPRINT.md` §2 (Morning Brief,
   Inputs) originally framed Morning Brief's own Confluence input as
   scoped to "the trader's own Watchlist instruments" only. This Sprint's
   own Mission explicitly directs Morning Brief to consume Dashboard's
   Decision Center output directly — which is scoped to Watchlist ∪ open
   Positions (S1-019). Resolved by following the Mission's own explicit,
   more specific integration instruction: Morning Brief's instrument scope
   mirrors Dashboard's exactly (Watchlist ∪ Positions), a disclosed,
   deliberate refinement of the Blueprint's own slightly narrower original
   framing, not a silent contradiction of it — both documents remain
   otherwise unmodified.
3. **Which contributor's evidence anchors the "why"/"confidence" text per
   entry** — resolved as the same lead contributor (`topContributors[0]`,
   already the highest-`detectionConfidence` entry) S1-019 already
   selected; no new selection logic.
