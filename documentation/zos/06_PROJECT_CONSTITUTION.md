# Purpose

This Constitution defines the mandatory engineering *process* rules
governing the Zenith project. These rules are binding for every
contributor, whether human or AI.

This document governs engineering process authority only (architecture
approval, sprint authority, scope control, reporting). It is a peer to,
not superseded by or superseding, `24_ZENITH_PRODUCT_CONSTITUTION.md`,
which governs product philosophy, trader psychology, design, and AI
persona. See that document's own §0.1 for the full disambiguation.

# Constitutional Rules

## Rule 1 --- Architecture Authority

Only the Architecture Team may approve or modify project architecture.

## Rule 2 --- Sprint Authority

Implementation begins only after an approved Sprint Brief.

## Rule 3 --- Scope Control

Implementation must remain within the approved sprint scope.

## Rule 4 --- No Unauthorized Decisions

Implementation engineers must stop and request guidance whenever an
architectural or product decision is required.

## Rule 5 --- Documentation First

Any architectural decision must be documented before implementation.

## Rule 6 --- Single Source of Truth

ZOS is the authoritative engineering knowledge base.

## Rule 7 --- Reporting

Every completed sprint must end with a structured completion report.

## Rule 8 --- Facts Over Assumptions

Reports must clearly distinguish FACTS, INFERENCES, and ASSUMPTIONS.

## Rule 9 --- Code Quality

All code must comply with approved coding standards and project
conventions.

## Rule 10 --- Continuous Improvement

Every approved sprint should improve the project without degrading
architecture or maintainability.

# Enforcement

Violations of these rules require Architecture Team review before work
may continue.
