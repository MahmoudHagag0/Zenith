# Purpose

This is not a UX flow, a user flow, or a screen flow — those exist at
`M4-002_SCREEN_ARCHITECTURE_BLUEPRINT.md`, unmodified by this document.
This is the **behavioral and psychological journey** of a professional
trader across a trading lifecycle — a day, a week, a decision, an
emotion — and how Zenith's already-approved architecture serves each
moment of it. No new product area, screen, navigation change, or visual
detail is introduced. Every claim traces to Constitution §6–§9, ZXL, or
D1-001 — this document applies already-approved psychology across
*time*, which none of those documents did explicitly.

# 1. Zenith Behavioral Philosophy

**Why traders return:** not because the app pulls them back, but
because each visit has previously, reliably, made their thinking
clearer — the same trust-calibration mechanism already established at
Constitution §6.3 and D1-001 §6 (Lee & See, trust in automation):
trust accrues from consistent, calibrated performance, not from any
single persuasive session.

**Why trust is stronger than engagement:** engagement metrics (session
count, time-on-app) can rise while decision quality falls — a product
optimizing for the former at the expense of the latter would directly
violate Zenith's own Mission (Constitution §2). Trust, by contrast, is
only built by the product being *right to rely on*, which requires
Zenith to sometimes want a trader to leave quickly (Dashboard answered
the question, nothing more needed) — a healthy visit is not always a
long one.

**Why value beats addiction:** addiction is maintained by intermittent,
unpredictable reward (D1-001 §8, Skinner's variable-ratio research) —
a mechanism Constitution §7.5/§12.3 already permanently forecloses.
Value is maintained by the trader's own observed decision quality
improving over time, which requires no unpredictability at all — the
opposite mechanism.

**Habit, Routine, Dependency, Addiction — stated precisely, since the
words are often used interchangeably but are not the same thing:**

| Term | Definition | Zenith's relationship to it |
|---|---|---|
| **Habit** | An automatic behavior triggered by a stable, low-friction cue, requiring little conscious deliberation (D1-001 §8, Lally et al.) | Neutral — can be healthy or unhealthy depending on what is reinforced. Zenith deliberately engineers *specific* habits (§7 below). |
| **Routine** | A deliberately structured sequence of habits performed regularly for a stated purpose, still under the trader's own conscious control | The target state: a trader's own morning routine, review routine, etc., with Zenith as one disciplined step within it — not the routine's owner. |
| **Dependency** | Reliance on a tool for a capability the person has not built themselves | Not Zenith's goal in the judgment sense (Zenith explains *why*, per Constitution §4.2, so the trader's own judgment grows) — but a healthy, *bounded* dependency on Zenith as a consistent evidence source (the way a professional depends on a reliable data terminal) is acceptable and expected. |
| **Addiction** | Compulsive engagement maintained by variable/unpredictable reward, continuing despite the person's own awareness of harm | Explicitly, permanently excluded (Constitution §3.2, §7.5, §12.3) — no streak, no badge, no unpredictable notification cadence, no variable-reward mechanic anywhere in this document. |

**Stated clearly, as required:** Zenith is designed to build the second
row of that table — healthy professional habits assembling into a
trader's own routine — never the fourth.

# 2. Complete Daily Trader Journey

Each stage: mindset, emotional state, primary questions, desired
psychological outcome, how Zenith helps, responsible screen (per
`M4-002`, unmodified).

## 2.1 Before Market Open

- **Mindset:** highest cognitive capacity of the day, lowest context for
  *today specifically* (Constitution §9.1).
- **Emotional state:** neutral-to-anticipatory; not yet activated by
  live price movement.
- **Primary questions:** "What changed since I last looked? Am I
  decision-ready?"
- **Desired outcome:** oriented and briefed before any raw data is seen
  (Constitution §12.1).
- **How Zenith helps:** synthesis before volume — the exact
  Constitution §10.2 Dashboard/Morning Brief Success Criteria.
- **Screen:** Dashboard, then Morning Brief.

## 2.2 Market Open

- **Mindset:** shift from preparation to live attention.
- **Emotional state:** alertness, some activation as real price action
  begins — a natural, not-yet-problematic arousal (Constitution §6.6
  concerns *acute*, decision-distorting arousal specifically, not all
  alertness).
- **Primary questions:** "Has anything I reviewed pre-open changed?"
- **Desired outcome:** confirmation, not re-analysis from scratch — the
  trader should not have to redo Before-Market-Open's own work.
- **How Zenith helps:** Watchlist's own aggregate tracked-state view
  (D1-005 §2.2) surfaces only what changed, not the full evidentiary
  base again.
- **Screen:** Watchlist, Alerts (if any triggered).

## 2.3 Market Monitoring

- **Mindset:** sustained, lower-intensity attention across the trading
  session.
- **Emotional state:** steady, ideally — the state Zenith's Calm
  Interface Philosophy (Constitution §5.2) exists to protect across
  this specific, longest stage of the day.
- **Primary questions:** "Is there anything requiring my attention right
  now?"
- **Desired outcome:** confidence that *absence* of an alert means
  absence of a genuine condition, not absence of monitoring (ZXL §7.1,
  reducing FOMO).
- **How Zenith helps:** Alerts surfaces only genuine, disclosed
  evidentiary change (Constitution §10.2, Alerts) — silence is
  meaningful, not ambiguous.
- **Screen:** Watchlist, Alerts.

## 2.4 Opportunity Detection

- **Mindset:** actively building understanding of a specific,
  emerging situation.
- **Emotional state:** rising interest/urgency-adjacent arousal — the
  exact moment Constitution §5.2's anti-manufactured-urgency principle
  matters most, since the trader's *own* arousal is rising regardless
  of what Zenith does.
- **Primary questions:** "What is the evidence here, specifically?"
- **Desired outcome:** the trader's own rising interest is met with
  calm, complete evidence — never amplified by Zenith's own presentation
  (D1-001 §7, D1-002 §6).
- **How Zenith helps:** Story Before Chart (Constitution §12.1) —
  synthesized evidence first, keeping analysis anchored rather than
  reactive to whatever caught the eye first (§12.1's own confirmation-
  bias rationale).
- **Screen:** Watchlist drill-in, Calendar/News, AI Workspace.

## 2.5 Decision Preparation

- **Mindset:** the highest-stakes, most emotionally-loaded moment in
  the entire journey (Constitution §9.3).
- **Emotional state:** whatever it is, is not Zenith's to amplify or
  suppress — only to accompany with maximal clarity (Constitution
  §9.3).
- **Primary questions:** "What is my confidence, specifically? What
  don't I know? Is 'no trade' actually the right read here?"
- **Desired outcome:** a decision made *with* full disclosed evidence,
  never pressured toward any particular action — including no pressure
  against inaction (Constitution §3.2, §12.4).
- **How Zenith helps:** the four-part Confidence taxonomy and equal-
  prominence uncertainty disclosure (Constitution §6.5, §12.6–12.7) —
  this is the single moment those mechanisms exist for.
- **Screen:** Watchlist/Portfolio drill-in, AI Workspace.

## 2.6 Trade Execution

- **Mindset:** committing to the decision prepared in §2.5.
- **Emotional state:** ideally calm, decisive — a natural consequence
  of §2.5 having been done well, not a separate state Zenith manages.
- **Primary questions:** N/A within Zenith itself.
- **Desired outcome:** N/A within Zenith itself.
- **How Zenith helps:** **nothing at this exact stage, by design.**
  Zenith is not an execution venue (Constitution §9.4, §10.2) — this
  stage happens at the trader's own broker/exchange, entirely outside
  Zenith's product surface. This is stated explicitly so no future
  screen is ever proposed for it without a separate, explicit business-
  model decision (as already noted at `M4-002` §6).
- **Screen:** none (outside Zenith).

## 2.7 Trade Monitoring

- **Mindset:** tracking a now-open position's own evolving state.
- **Emotional state:** the disposition-effect danger zone (D1-001 §7,
  Shefrin & Statman) — the pull to hold a loser too long or exit a
  winner too early lives here specifically.
- **Primary questions:** "What is this position's own current
  evidentiary state — not my feeling about it?"
- **Desired outcome:** neutral disclosure regardless of favorability
  (Constitution §7.3, §3.3) — the direct countermeasure to the
  disposition effect's own emotional pull.
- **How Zenith helps:** Portfolio's own calm, identical-treatment-
  regardless-of-favorability presentation (D1-005 §2.3).
- **Screen:** Portfolio.

## 2.8 Trade Closing

- **Mindset:** the decision to exit is itself a second Decision
  Preparation moment (§2.5), not merely a formality.
- **Emotional state:** relief, disappointment, or something more
  complex — again, not Zenith's to shape.
- **Primary questions:** same as §2.5, applied to exiting rather than
  entering.
- **Desired outcome:** same as §2.5.
- **How Zenith helps:** same mechanisms as §2.5, reused rather than a
  separate "exit-specific" pattern invented — consistency (Constitution
  §5.4).
- **Screen:** Portfolio, AI Workspace.

## 2.9 Reflection

- **Mindset:** shifting from live decision-making to review.
- **Emotional state:** should be calm regardless of outcome —
  Constitution §3.3's non-punitive framing applies most directly here.
- **Primary questions:** "What did I actually know at the time? What
  would I do differently, if anything?"
- **Desired outcome:** neutral, evidence-based reflection (Constitution
  §7.3) — a losing trade reviewed with the same calm as a winning one.
- **How Zenith helps:** Trading Journal's own explicit non-punitive
  design (Constitution §10.2, Trading Journal).
- **Screen:** Trading Journal.

## 2.10 End-of-Day Review

- **Mindset:** closing the day's own loop, not opening a new one.
- **Emotional state:** should trend toward settled, whatever the day's
  own outcomes were.
- **Primary questions:** "Is there a real process pattern across today's
  decisions, or is this one data point?"
- **Desired outcome:** the Learning phase of the user journey
  (Constitution §9.6) — Review becomes the cue for better future
  Analysis, not a one-off report card.
- **How Zenith helps:** Trading Journal's own cross-entry pattern
  identification (Constitution §10.2, Trading Journal Success
  Criteria).
- **Screen:** Trading Journal, Portfolio.

## 2.11 Logout

- **Mindset:** a session boundary, not a Decision Flow question at all
  (per `M4-002` §4.11).
- **Emotional state:** N/A.
- **Primary questions:** N/A.
- **Desired outcome:** N/A — logout is deliberately not a moment Zenith
  attaches any psychological weight to (no "come back soon," no guilt,
  no streak-loss framing).
- **How Zenith helps:** by doing nothing beyond ending the session
  cleanly.
- **Screen:** Login (transition only).

# 3. Weekly Journey

| Day | Cognitive need | Zenith's role |
|---|---|---|
| **Monday** | Re-orientation after a multi-day evidence gap; highest planning load of the week | Morning Brief carries more relative weight than a mid-week Monday — more genuinely changed since last session (Constitution §9.1 applied at weekly, not just daily, granularity). |
| **Midweek** | Steady-state monitoring/execution rhythm — the daily journey (§2) repeating with minimal re-orientation cost | Watchlist/Alerts/Portfolio carry the routine; Morning Brief's own relative weight is lower (less has changed since yesterday). |
| **Friday** | Closing-the-week judgment: exposure into the weekend, a natural evaluation point | Portfolio review with weekend-exposure awareness (a factual disclosure, e.g. market-closed state, not a manufactured "act before the weekend" urgency — Constitution §5.2 applies identically at weekly cadence). COT & Reports' own weekly-publication cadence (L1-004) naturally lands review here too. |
| **Weekend** | Learning and planning, not monitoring — markets are closed for most tracked instrument classes | Trading Journal (full week-in-review, Constitution §9.6's Learning phase at weekly granularity) and Calendar/News (the coming week's scheduled events, prepared calmly rather than reactively on Monday morning). |

# 4. Decision Journey

For each major decision type: what the trader thinks, what usually goes
wrong, which named bias, how Zenith intervenes (already-approved
mechanisms only), expected outcome.

## 4.1 Entering a Position

- **What the trader thinks:** "This looks like a real opportunity."
- **What usually goes wrong:** the eye catches a dramatic chart pattern
  first and reasons backward to justify it.
- **Bias:** confirmation bias (Constitution §12.1, citing Nickerson).
- **Zenith's intervention:** Story Before Chart — synthesized evidence
  is disclosed before any chart, anchoring interpretation to evidence
  rather than to whatever the eye caught first.
- **Expected outcome:** a decision traceable to disclosed evidence, not
  to visual impression.

## 4.2 Holding Through a Drawdown

- **What the trader thinks:** "It will come back."
- **What usually goes wrong:** a losing position is held longer than
  its own evidence supports, because realizing the loss is more painful
  than an unrealized one of the same size.
- **Bias:** disposition effect / loss aversion (D1-001 §7, Shefrin &
  Statman; Kahneman & Tversky).
- **Zenith's intervention:** the position's own evidentiary state is
  disclosed with the exact same calm, undramatized treatment regardless
  of favorability (Constitution §7.3) — no design choice makes holding
  *feel* easier or harder than the evidence itself supports.
- **Expected outcome:** the hold/exit decision is made on current
  evidence, not on the emotional cost of realizing a loss.

## 4.3 Exiting a Winning Position Too Early

- **What the trader thinks:** "I should lock this in before it
  reverses."
- **What usually goes wrong:** the same disposition-effect asymmetry as
  §4.2, in the opposite direction.
- **Bias:** disposition effect (same source as §4.2).
- **Zenith's intervention:** identical to §4.2 — one mechanism serves
  both directions of the same bias, per Constitution §5.4 (consistency).
- **Expected outcome:** same as §4.2.

## 4.4 Increasing Position Size After a Win

- **What the trader thinks:** "I'm reading this market well right now."
- **What usually goes wrong:** a recent string of correct calls is
  mistaken for durably higher skill/confidence than the evidence
  actually supports.
- **Bias:** overconfidence (ZXL §7.4).
- **Zenith's intervention:** the four-part Confidence taxonomy is never
  collapsed into one number (Constitution §6.5, §12.6) — a trader
  cannot mistake a strong pattern-match for a strong regime-context
  read, which is precisely the conflation overconfidence exploits.
- **Expected outcome:** sizing decisions grounded in the current
  decision's own specific confidence breakdown, not in recent streak.

## 4.5 Taking a Trade Despite Low Confidence

- **What the trader thinks:** "I don't want to miss this."
- **What usually goes wrong:** fear of missing out overrides a
  genuinely low-confidence or absent reading.
- **Bias:** FOMO (ZXL §7.1).
- **Zenith's intervention:** "no clear opportunity" is treated as a
  fully legitimate, equally well-supported outcome (Constitution §12.4)
  — never minimized, hidden, or presented as a product gap the trader
  should feel bad about trusting.
- **Expected outcome:** a low-confidence reading is experienced as
  honest information, not as a frustrating absence.

## 4.6 Re-Entering After a Loss ("Making It Back")

- **What the trader thinks:** "I need to make that back."
- **What usually goes wrong:** frustration/urgency from the prior loss
  is carried directly into the next decision.
- **Bias:** revenge trading (ZXL §7.5).
- **Zenith's intervention:** the new decision's own evidence is
  presented with the exact same calm register as any other — nothing in
  Zenith's own presentation amplifies the emotional charge already
  present from the prior loss (Constitution §6.6).
- **Expected outcome:** the new decision stands on its own evidence,
  not on the emotional debt of the last one.

## 4.7 Analysis Paralysis (Failing to Decide at All)

- **What the trader thinks:** "I need to check one more thing before
  I'm sure."
- **What usually goes wrong:** the full evidentiary base is consulted
  repeatedly without ever synthesizing into a decision.
- **Bias:** analysis paralysis (ZXL §7.3).
- **Zenith's intervention:** Progressive Disclosure (D1-005 §3) gives a
  synthesized answer first, with deeper evidence reached only
  deliberately — the trader is never handed the full evidentiary base
  and left to build the synthesis themselves.
- **Expected outcome:** a decision reached in a bounded amount of
  review, not an open-ended one.

# 5. Emotional Journey

For each emotion: when it typically arises, and how Zenith responds —
**without manipulating it** (never amplifying, never suppressing,
never using the emotion as an engagement lever).

| Emotion | When it arises | Zenith's response |
|---|---|---|
| **Confidence** | After a well-evidenced decision or a correct recent call | Disclosed factually via the Confidence taxonomy (Constitution §6.5) — never inflated by tone, never a "great call!" congratulatory register (ZXL §8, never exciting). |
| **Fear** | Approaching a decision, or during a drawdown (§4.2) | Met with maximal evidence clarity, zero design pressure toward or against any action (Constitution §9.3) — fear is not soothed with reassurance language, it is met with disclosed fact. |
| **Greed** | Opportunity Detection (§2.4), a strong-looking setup | Met with the same calm, non-amplifying presentation as any other reading (Constitution §5.2) — Zenith does not "match the trader's excitement." |
| **Frustration** | After a loss, or an unclear/absent reading | Met with neutral disclosure (Constitution §7.3) — never apologized for, never over-explained defensively, just disclosed as plainly as any other state. |
| **Fatigue** | Later in a long session or a long week | Met by *not* compounding it — no re-orientation cost forced on the tenth decision that the first didn't have (Constitution §6.2); this is a structural response (already built into navigation/consistency, `M4-002` §9), not a message shown to the trader. |
| **Overconfidence** | After a winning streak (§4.4) | Met by the taxonomy's own resistance to collapse (§4.4) — the intervention is structural, not a cautionary message ("be careful!") which would itself be a form of emotional steering Zenith does not do. |
| **Revenge mindset** | Immediately after a loss (§4.6) | Met by identical calm treatment of the next decision's own evidence (§4.6) — again structural, not a scolding or a cooldown-timer gimmick, which would itself be a manipulative pattern. |

**The common thread, stated once:** every response above is a
*structural* property of how evidence is disclosed (consistent,
calm, non-amplifying), never a *message* crafted to make the trader
feel a particular way. Zenith influences emotional experience only as
a side effect of being honest and calm — never as a direct lever.

# 6. Cognitive Journey

| Burden | Description | Which screens reduce it, and how |
|---|---|---|
| **Working memory load** | Only ~4 actively-reasoned items held at once (D1-001 §1, Cowan 2001) | Dashboard/Morning Brief do the synthesis so the trader holds one conclusion, not the full evidentiary base, in mind (Constitution §12.2). |
| **Attention shifts** | Full attention is a scarce, sequential resource (D1-001 §2, Broadbent) | Exactly one Primary Attention element per screen (D1-002 §1.1) means no screen forces a split-attention decision about where to look first. |
| **Decision fatigue** | Later decisions in a session risk lower-effort defaults (Constitution §6.2) | Consistent navigation and presentation (`M4-002` §9) mean the tenth decision of the day costs no more re-orientation than the first. |
| **Information overload** | More visible information is not more understanding | Progressive Disclosure (D1-005 §3) — Watchlist/Portfolio show aggregate state first, individual depth only on request. |
| **Pattern recognition** | Traders build real skill by recognizing recurring situations over time | Trading Journal's own cross-entry review (Constitution §10.2) is the one screen whose explicit purpose is surfacing genuine patterns across many past decisions, not just recalling one. |
| **Learning loop** | Skill only compounds if Review actually informs future Analysis (Constitution §9.6) | Trading Journal → next session's Morning Brief/Decision Preparation is the loop — Reflection is the cue, not the end, of the cycle (D1-001 §8, cue-routine-reward, applied to the *disciplined* routine specifically). |

# 7. Habit Formation

Each habit below is deliberately designed around D1-001 §8's own
finding: sustainable habits form through low-friction repetition of a
cue-routine-reward loop — and Constitution §7.5/§12.3's own permanent
rule that the *reward* must never be activity itself.

| Habit | Cue | Routine | Reward (never activity-based) |
|---|---|---|---|
| **Morning routine** | Start of trading day | Dashboard → Morning Brief | A genuinely faster, calmer path to "am I decision-ready" than reconstructing it manually — the reward is *time and clarity saved*, not a completed checklist. |
| **Mid-session check** | A natural pause in active monitoring | Watchlist/Alerts glance | Confirmation that nothing was missed (§2.3) — the reward is *reduced anxiety*, not a "checked in" streak. |
| **Post-trade review** | A position closes | Trading Journal entry | A clearer record of one's own reasoning available for the *next* similar decision — the reward is *future decision quality*, not "entry logged." |
| **End-of-day review** | Session end | Trading Journal + Portfolio | A genuine pattern noticed across the day's decisions, if one exists — never a forced "you reviewed today!" acknowledgment if nothing meaningful was found. |
| **Weekend review** | Week end | Trading Journal (week view) + Calendar/News (week ahead) | Entering next week with real, disclosed context — the reward is *preparedness*, not attendance. |

**Explicitly excluded, per this task's own instruction and Constitution
§3.2/§7.5/§12.3:** no daily-open streak, no login-count badge, no
"you're on a 5-day streak!" messaging of any kind. A trader who
correctly decides not to open Zenith on a given day (nothing has
changed, no decision is pending) has used the product exactly as
intended — that is not a failure state to discourage.

# 8. Screen Sequence Rationale

Uses the already-approved order from `M4-002` §1/§3.1 (Decision Flow-
derived) — **not** re-derived or changed here. Restated with its
psychological rationale made explicit per transition:

1. **Dashboard → Morning Brief.** Dashboard answers "am I decision-
   ready" (Q1) in the fewest possible steps; Morning Brief then answers
   "why" (Q2) for a trader who wants the fuller synthesis. The
   transition exists because *knowing the readiness state first*
   changes how the "why" is received — as elaboration, not as raw
   material to synthesize alone.
2. **Morning Brief → Watchlist/Portfolio.** Having been briefed on
   what changed, the trader now asks "where, specifically" (Q3) —
   Watchlist for tracked-but-not-held instruments, Portfolio for held
   ones. The transition exists because Q2's answer names specific
   instruments Q3 then lets the trader inspect directly, rather than
   forcing a second search.
3. **Watchlist/Portfolio → AI Workspace / Trading Journal.** Once a
   specific instrument's evidence is in view, the trader may need a
   clarifying question answered (AI Workspace) or want to record/
   consult reasoning about it (Trading Journal, Q5). The transition
   exists because these are genuinely later questions in the Decision
   Flow — asking "what should I learn from this" only makes sense once
   "where, specifically" has already been answered.

**On "Decision Companion":** no screen by this name exists in
`M4-002`'s 11-screen inventory. This document treats it as referring to
Dashboard's own existing Decision Center component (ZXL §9.1,
`DASH-002`) — the psychological role a "decision companion" concept
would describe is already filled by Dashboard itself, not a separate
screen. This resolution is stated explicitly rather than assumed
silently (see §11 below).

# 9. "Why Zenith First?"

A professional trader opens Zenith before charts for the same reason a
professional pilot reviews a briefing before touching the controls: **it
is the fastest, calmest path to an accurate answer to "where do things
stand," and every subsequent action is better-informed for having asked
that question first.** This is a value claim, not a behavioral hook:

- **Not notifications** — Zenith does not push the trader back in; the
  habit (§7) is cued by the trader's own session start, not by an
  external interrupt engineered to reclaim attention.
- **Not fear** — nothing about not opening Zenith first is framed as
  risk, loss, or missing out (directly foreclosing ZXL §7.1's own named
  failure mode).
- **Not artificial engagement** — there is no reward for opening Zenith
  itself (§7) — only for what the trader gains once inside: a
  genuinely faster, more complete, calmer starting point than assembling
  the same picture manually from raw charts and headlines.

The honest test of this claim: a trader who opens Zenith first should
be able to state *why* — not "habit" alone, but a specific, real time-
or-clarity saving they have personally experienced repeatedly. If that
test fails, this document's own premise fails with it, and this section
would need revision, not defense.

# 10. Success Criteria

Stated in the same observable, non-visual-design-dependent form
Constitution §10.1 already requires for every screen's own Success
Criteria — applied here at the behavioral-journey level:

1. **Fewer impulsive decisions** — a trader's own Trading Journal shows
   a declining rate of entries later self-identified as made without
   reviewed evidence.
2. **Increased self-awareness** — a trader can articulate their own
   recurring decision patterns (a real bias, a real strength) sourced
   from Trading Journal review, not from a generic list.
3. **Consistent trade review** — Trading Journal entries exist for a
   genuinely high proportion of closed positions, driven by the habit
   loop in §7, not by a reminder mechanic.
4. **Calibrated trust** — a trader can correctly state, for a given
   reading, both its confidence *and* its disclosed limitations,
   without either being overstated (Constitution §6.5, §12.7).
5. **Natural session start** — a trader opens Zenith first not because
   of an external prompt, but because they can articulate the specific
   value doing so has repeatedly provided (§9's own honesty test).
6. **No manufactured engagement artifacts exist** — no streak count, no
   badge, no unpredictable notification cadence appears anywhere in the
   product, verifiable by inspection, not merely by policy statement.

# 11. Ambiguities Discovered

1. **"Decision Companion"** (named in this task's own §8 example list)
   is not one of `M4-002`'s 11 approved screens. Resolved conservatively
   as referring to Dashboard's existing Decision Center component
   (ZXL §9.1, `DASH-002`) — not a new screen. If a genuinely separate
   "Decision Companion" screen is intended, that is a new product-area
   proposal requiring Constitution §10.1 approval, not something this
   document can authorize.
2. **The example screen-sequence order in this task's own §8** ("Morning
   Brief, Dashboard, Watchlist, Portfolio...") places Morning Brief
   before Dashboard — the reverse of `M4-002`'s already-approved order
   (Dashboard first, answering Q1; Morning Brief second, answering Q2).
   Per this task's own explicit constraint ("Do NOT redesign
   navigation"), §8 above justifies the *already-approved* order, not
   the example list's literal sequence.
3. **"Trade Execution" as a journey stage** required stating explicitly
   that Zenith has no role there (§2.6) — this is a direct, load-bearing
   consequence of Constitution §9.4/§10.2's existing execution-venue
   boundary, not a new restriction invented here, but worth flagging
   since the task's own example timeline could otherwise be read as
   implying Zenith participates in execution.
